# ADR 0004: Mobile build strategy через Expo

- Статус: accepted
- Дата: 2026-04-13

## Контекст

Команда хочет избежать тяжелых локальных сборок мобильного приложения и использовать удаленную инфраструктуру сборки.

## Решение

Использовать `React Native + Expo` и собирать мобильные приложения через `Expo/EAS`.

Принципы:

- локальная машина используется для разработки, dev-сборок и smoke tests;
- cloud build используется для iOS и Android release artifacts;
- GitHub Actions может инициировать EAS build для release веток или tagged releases;
- подписи, секреты и credentials хранятся вне репозитория.

## Последствия

Плюсы:

- меньше требований к локальным машинам;
- одинаковый процесс сборки для команды;
- проще организовать release pipeline.

Минусы:

- появляется зависимость от внешнего build provider;
- нужен контроль версий Expo SDK и native dependencies.
